Reasons behind the designs

Homepage wireframe: The layout allows users to have options to navigate to other pages including links and sliders. The interface will include images to make it attractive and compliment the kind of information that is conveyed in the page. The left and right arrows will be buttons to allow the user to move the slider cards. There will be an autoplay slideshow for both the web design section and the blog posts section, the page will also include a new page which will show the user all the pages they are going to interact with by giving them a bit of information about the page. I chose this layout because it will keep the content in order and the user will have a clear understanding when they navigate the page that they are on the homepage because the interface will be very different from the one of other pages.

Blog posts wireframe: I chose this layout because the information will be neatly presented and legible with a simple and effortless navigation.

Planning documentation wireframe: More like the style guide page, it will include a documentation and reflection on the design process of the charts and reasons behind my design choices.

Styleguide wireframe: I chose the same layout as the blog postsâ€™ page because the page will include a lot of information that I want to keep legible and spaciously presented. It will include critical writing of my style choices and reasons behind the choices as well as how these choices have or had impacts on the design.

Wireframes wireframe: I went for the slider layout to keep the content in one place and let the user have time to look at the wireframes slide-by-slide. I chose this type of layout because there will not be too much text on the page, and the wireframes have enough information on their own. Then I will include text which briefly explains my user experience design choices.

Data-Art wireframe: The layout will be the same as the data visualizations page layout, this is because I want the content to be spacious and well presented. There will be illustrations and more information about them on the side so that the user does not have to navigate down the page to read the information.

Data Visualization wireframe: The layout will have illustrations with supporting captions and information on the opposite side, I chose it because it will present the content legibly, the page will be appealing, and easy to navigate .

Blog pages wireframe: The layout will be the same as the blog posts layout because I want to keep the consistency in the website.
