Reasons behind the designs

Homepage wireframe: The layout allows users to have options to navigate to other pages including links and sliders. The interface will include images to make it attractive. The left and right arrows will be buttons to allow the user to move the slider cards. I chose this layout because it will keep the content in order and the user will have a clear understanding when they navigate the page that they are on the homepage.

Blog posts wireframe: I chose this layout because the information will be neatly presented and readable with a simple navigation.

Reflections wireframe: More like the style guide page, it will include reflections on the design process and reasons behind my design choices.

Styleguide wireframe: I chose the same layout as the blog postsâ€™ page because the page will include a lot of information that I want to keep legible and spaciously presented. It will include critical writing of my style choices and reasons behind the choices as well as how these choices have or had impacts on the design.

Wireframes wireframe: I went for the slider layout to keep the content in one place and let the user have time to look at the wireframes slide-by-slide. I chose this type of layout because there will not be too much text on the page, and the wireframes have enough information on their own.

Data-Art wireframe: I have not thought about the layout of this page thoroughly. But surely the content will be spacious and well presented. There will be illustrations and more information about them.

Data Visualization wireframe: The layout will have illustrations with supporting captions, I chose it because it will present the content legibly, I might change it on the final design to a slider display to make it more appealing.
